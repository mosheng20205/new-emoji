# 易语言 UTF-8 字节集规则

易语言 IDE 源码中直接输入 emoji、部分特殊符号、部分扩展 Unicode 字符时，可能显示或保存为 `?`。因此易语言示例不要直接写 `"🚀"`、`"✅"` 这类源码字符串。

## 必须使用的写法

- 中文文本、emoji、特殊符号都先转为 UTF-8 字节集。
- 调用 DLL 时同时传入字节集指针和字节长度。
- 标题、按钮 emoji、按钮文字要分别准备字节集变量。
- `.DLL命令` 左侧命令名给易语言用户调用，必须中文化；右侧入口名保持真实 C++ 导出名，例如 `"EU_CreateButton"`。

## 示例字节集

```text
窗口标题 "✨ 易语言窗口" 的 UTF-8 字节：
{ 226, 156, 168, 32, 230, 152, 147, 232, 175, 173, 232, 168, 128, 231, 170, 151, 229, 143, 163 }

按钮 emoji "✅" 的 UTF-8 字节：
{ 226, 156, 133 }

按钮文字 "确认操作" 的 UTF-8 字节：
{ 231, 161, 174, 232, 174, 164, 230, 147, 141, 228, 189, 156 }
```

AI 生成易语言代码或说明时，应优先输出“字节集变量 + 长度”的形式，避免把 emoji 作为源码字符串写入。
