# 易语言 AI Prompt

```text
你正在使用 new_emoji.dll 为易语言生成 Windows 原生 UI。

请优先阅读：
1. docs/ai/e-language-encoding.md
2. DLL命令/易语言DLL命令.md
3. docs/ai/api_manifest.full.json
4. examples/templates/易语言/空白窗口.md

规则：
- 易语言源码中不要直接写 emoji 或特殊 Unicode 字符，否则可能变为问号。
- 所有中文和 emoji 文本必须使用 UTF-8 字节集 + 长度传给 DLL。
- .DLL命令 左侧命令名必须中文，右侧入口名保持 EU_ 导出名。
- 回调函数指针参数如果声明为 整数型，调用时必须写 到整数 (&回调子程序名)，不要直接传 &回调子程序名。
- 例如：设置标签页重排回调 (窗口句柄, Chrome标签页, 到整数 (&标签页重排回调))。
- 不要生成错误写法：设置标签页重排回调 (窗口句柄, Chrome标签页, &标签页重排回调)。
- 回调子程序的参数个数和顺序必须严格匹配对应 DLL 导出的真实回调签名，不能套用其它组件的参数模板。
- 尤其是功能按钮常用的 `EU_SetElementClickCallback` / `设置元素点击回调`：C++ 类型为 `void __stdcall callback(int element_id)`，易语言回调子程序只能写 1 个参数 `.参数 元素ID, 整数型`。
- 不要给 `设置元素点击回调` 的回调子程序添加 `数值`、`范围开始`、`范围结束` 等额外参数；多写参数会造成调用栈错位，点击功能按钮时可能直接闪退。
- 只有滑块、范围、表格、树等专用回调文档明确写了更多参数时，才按对应文档补充参数；生成前必须查 `DLL命令/易语言DLL命令.md` 或 C++ typedef。
- Win32 DLL 是易语言交付优先目标。
- 界面文案必须中文化，窗口标题、主按钮、核心展示项必须有 emoji，但 emoji 也必须以 UTF-8 字节集形式传入。
- 窗口和根容器首次打开时不能裁切控件，要保留至少 20px 逻辑余量。
- 编写的易语言代码文件用".e.txt"后缀，不要使用.e后缀，因为易语言的.e后缀文件是二进制文件你无法写入
- 易语言的DLL命令中如果参数类型是整数型，不要直接传入字节集参数，而是使用：取变量数据地址 (字节集)
```

Table 普通表格行级增删使用 `.DLL命令 添加表格行`、`插入表格行`、`删除表格行`、`清空表格行`，右侧 DLL 入口分别绑定 `EU_AddTableRow`、`EU_InsertTableRow`、`EU_DeleteTableRow`、`EU_ClearTableRows`；虚表数据由回调数据源维护。

Table 普通文本单元格双击编辑使用 `.DLL命令 设置表格双击编辑` 开启；列/单元格覆盖使用 `设置表格列双击编辑` / `设置表格单元格双击编辑`，编辑回调 `设置表格单元格编辑回调` 的文本参数是 UTF-8 字节指针 + 长度。

窗口图标：使用 `.DLL命令 设置窗口图标` 绑定 `EU_SetWindowIcon` 传入 UTF-8 `.ico` 路径字节集，或使用 `.DLL命令 从字节集设置窗口图标` 绑定 `EU_SetWindowIconFromBytes` 传入 `.ico` 文件完整字节集。

窗口圆角：使用 `.DLL命令 设置窗口圆角` 绑定 `EU_SetWindowRoundedCorners(hwnd, enabled, radius)` 设置 Windows 窗口外形圆角；支持 DWM 的系统会优先使用平滑圆角，Win10 会回退为 per-pixel alpha 分层窗口并通过 `UpdateLayeredWindow(ULW_ALPHA)` 提交。
